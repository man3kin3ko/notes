Серверные компоненты, использующиеся для управления внутренним состоянием кластера, называются Control Plane. Вспомогательные компоненты, развернутые на каждой ноде в ворклоаде, называются node components. Это kubelet, клиент API-сервера, container runtime (CRI) и kube-proxy

## etcd

Этот компонент используется для хранения конфигурации кластера, в том числе секретов. Инстансов etcd может быть несколько, поэтому лучшей практикой является создание отдельного CA для общения инстансов etcd друг с другом и для общения с ворклоад нодами через API.

По умолчанию секреты в etcd хранятся в открытом виде.

> **💡** При включении шифрования etcd секреты шифруются только при записи, поэтому после включения необходимо вручную зашифровать все имеющиеся секреты с помощью kubectl
